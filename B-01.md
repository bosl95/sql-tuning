# 쿼리 튜닝 과정

```sql
select
COUNT(case when programmer.hobby = 'yes' then 1 end) / COUNT(*) * 100 as YES,
COUNT(case when programmer.hobby = 'no' then 1 end) / COUNT(*) * 100 as NO
from programmer;
```

- hobby에 인덱스를 걸어주는 것은 불필요하다. (결국 모든 로우를 방문해야하기 때문)
